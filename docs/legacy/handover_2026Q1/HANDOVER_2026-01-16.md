## 1. Active Time 采集（Label Studio 端）

对应文件：

- `tools/ls_userscript.js`（Tampermonkey 脚本）
- `tools/cors_server.py`（日志接收服务，落盘到 `active_logs/`）

### 1.1 计时逻辑（关键点）

- **每秒累积**：当页面可见、且最近一次用户活动距今 < 15s（`IDLE_THRESHOLD`）时，`activeSeconds += 1`。
- **短暂切换标签页容忍**：页面隐藏 < 6s 不中断计时；隐藏 ≥ 6s 会重置“最近活动时间”，需要新的用户操作重新开始累计。
- **任务切换立即 flush**：独立 1s 任务检测器检测到 taskId 变化时，会把上一个任务的 `activeSeconds` 立即上报，并清零计数。
- **异步竞态保护**：flush 时会把 `activeSeconds` 复制到局部变量（`reportSeconds`），避免异步请求过程中被清零导致上报 0s。

### 1.2 如何验证是否在工作

在 Label Studio 的浏览器控制台中应看到类似日志：

- `[TASK_INIT] 初始化任务ID: ...`
- `[TASK_SWITCH] ... -> ...，立即上报...s`
- `[FLUSH] 已立即上报任务 ... 的 ...s 活动时间`

落盘位置：`active_logs/active_times_YYYY-MM-DD.jsonl`。

### 1.3 安全与配置（不写死在仓库）

- 服务端可通过环境变量 `HOHONET_LOG_TOKEN` 开启 token 校验（见 `tools/cors_server.py`）。
- 客户端 Token 通过浏览器 `localStorage` 设置（脚本读取 `HOHONET_LOG_TOKEN`）；不要把真实值写进文档或提交到 git。
- 服务器地址同样通过 `localStorage` 的 `HOHONET_HELPER_BASE_URL` 配置，避免硬编码。

## 2. 分析与可视化工具链（推荐入口）

目标：从 Label Studio 导出的标注 JSON 得到 **long-format 质量表**（task×annotator），并在 notebook 中做交互式可视化与严苛排障。

推荐流程：

1. `tools/analyze_quality.py`：导出质量 CSV（包含门控结果、scope、active_time 等字段）。
2. `tools/aggregate_analysis.py`：把多次/多条件输出合并成一份可比较的总表。
3. `tools/viz_quality_analysis.ipynb`：交互式可视化与诊断（函数来自 `tools/viz_quality_utils.py`）。
4. 可选：`tools/save_quality_figures.py` 批量导出图片与表格。
