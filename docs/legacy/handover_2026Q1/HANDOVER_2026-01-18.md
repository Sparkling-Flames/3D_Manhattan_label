01-18 版本聚焦一件事：**IAA/r_u 计算的门控修复与脚本边界澄清**。

## 1. 关键 Bug 修复

### 1.1 LOO 门控污染（BUG #1）

**问题**：`task_user_poly`（用于共识/LOO 计算）在填充时未检查 `layout_used`，导致被 L1 门控排除的标注（如坐标解析失败、覆盖率不足）仍然参与 IAA/r_u 计算。

**影响**：16 行 `layout_used=False` 的数据仍有 `iou_to_consensus_loo` 值，污染了可靠性指标。

**修复**：`task_user_poly` 填充条件增加 `layout_used==True`：

```python
# 修复后的逻辑 (tools/analyze_quality.py)
if not is_oos and not scope_missing and layout_used:
    task_user_poly[task_id][uid] = poly_points
```

**验证**：`quality_report_20260118.csv` 中所有 `layout_used=False` 的行 `iou_to_consensus_loo` 均为 NaN。

### 1.2 scope_missing 口径污染（BUG #2）

**问题**：v2 模式下，当 scope 字段缺失但存在旧版 `quality` 文本时，代码会回退到 legacy 逻辑推断 `is_normal`，导致 9 行 scope=NA 的数据被判定为 `is_normal=True`。

**影响**：本应标记为"未知"的行被错误纳入 In-scope 计算。

**修复**：v2 模式下无结构化字段时不回退 legacy，直接返回 `scope_missing=True, is_normal=None`：

```python
# 修复后的逻辑 (tools/analyze_quality.py, parse_quality_flags_v2)
if scope_str is None:
    scope_missing = True
    is_oos = None
    is_normal = None
    # 不再回退 legacy
```

**验证**：`quality_report_20260118.csv` 中所有 `scope_missing=True` 的行 `is_normal` 均为空。

### 1.3 scope_missing 列输出

**问题**：CSV 输出未包含 `scope_missing` 列，下游需要重复推导。

**修复**：`scope_missing` 现在始终包含在 CSV 输出中。

## 2. 脚本边界澄清

遵循 **Krippendorff (可靠性理论)** 和 **Artstein-Poesio (IAA 设计)** 的"单一事实源 + 可审计"原则：

### 2.1 上游/计算层 (`analyze_quality.py`)

**职责**：从 Label Studio JSON 解析数据、计算所有指标、应用门控规则、生成 CSV。

**权威字段**（下游只读取不重算）：
- `scope_missing`：scope 字段是否缺失
- `is_oos` / `is_normal`：scope 派生的 OOS/In-scope 判定（tri-state）
- `layout_used` / `layout_gate_reason`：L1 标准 layout 指标门控
- `iou_to_consensus_loo` / `consensus_uid_loo`：L3 可靠性指标

### 2.2 下游/展示层 (`visualize_output.ipynb`)

**职责**：读取 CSV、做 sanity check（一致性校验）、生成论文图表。

**派生字段**（仅用于展示/sanity check）：
- `scope_clean` / `is_oos_clean` / `is_normal_clean`：对 scope 列做标准化后重新判定
- `data_valid_for_main` / `data_valid_for_reliability`：论文图表的样本掩码

### 2.3 门控层次

| 层级 | 门控字段 | 用途 |
|------|----------|------|
| **L1 标准 layout 指标** | `layout_used` | 控制 `layout_2d_iou` / `layout_3d_iou` 等 HoHoNet 风格指标 |
| **L3 可靠性/LOO 共识** | `iou_to_consensus_loo.notna()` | 控制该行是否参与 IAA/r_u 计算 |

**关键设计**：L1 和 L3 是独立但协调的——L3 需要 L1 通过作为前置条件。
