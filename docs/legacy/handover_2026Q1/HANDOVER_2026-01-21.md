## 1. 关键口径更新：P0-1（mismatch 仅做诊断/分层）

### 1.1 标准 layout 指标（L1）不再因为点数不匹配而 gate

- **变更点**：点数不匹配（`pred_n_pairs != ann_n_pairs`）不再导致 `layout_used=False`。
- **原因**：layout 2D/3D IoU 与 layout-rendered depth 属于“整体几何质量”指标，不依赖 1-1 角点配对；硬 gate 会系统性排除“强纠错/高工作量”样本，造成统计偏差。
- **落地位置**：上游核心逻辑在 [tools/analyze_quality.py](../tools/analyze_quality.py) 的 `compute_layout_standard_metrics()`。

> 注意：`layout_gate_reason` 仍可能为 `low_coverage` / `normalize_failed` / `odd_points` / `x_inconsistent` / `exception` 等；深度指标本身缺失时会用 `depth_failed` 标记，但 `layout_used` 仍可能为 True（2D/3D IoU 仍可用）。

### 1.2 Pointwise RMSE（L2）仍保留 mismatch gate（技术必须）

- **保留点**：`compute_pointwise_rmse_cyclic()` 需要 1-1 对应的 corner pairs。
- **表现**：当点数不匹配时，`pointwise_rmse_used=False` 且 `pointwise_gate_reason="n_pairs_mismatch"`。

### 1.3 notebook 侧的 mismatch 统计口径

- **要求**：mismatch 不依赖 `layout_gate_reason` 推断。
- **实现**：以 CSV 的 `pred_n_pairs` / `ann_n_pairs` 字段为准：
  - `n_pairs_mismatch := (pred_n_pairs != ann_n_pairs)`
