# HANDOVER_2026-01-26: Database Rigor & Selection Bias Solution

## 1. 核心改进：解决 Selection Bias (选择偏误)

在之前的版本中，如果预测结果（Prediction）无法与真值（Annotation）达成角点配对（如点数不匹配），IoU 会因为无法计算而变为 `NaN`，从而在统计平均分时被自动忽略。

- **风险**：这会导致“只在模型预测成功的简单样本上算分”，虚高指标。
- **2.0 方案**：在 `analyze_quality.py` 中将 IoU 初始值设为 `0.0`。
- **效果**：如果模型严重失败（无法形成有效多边形），该样本将被记为 0 分而非剔除。这使得性能报告（Performance Report）具有完整的科学客观性。

## 2. 核心改进：鲁棒追加 (Robust Append)

解决了 CSV 文件在多次追加过程中的“Horizontal Shifting (列位移)”损坏风险。

- **问题**：新版脚本可能增加了新字段，直接追加到旧版 CSV 会导致列对不齐，数据彻底报废。
- **2.0 方案 (Header Alignment)**：
    - 追加前读取现有 CSV 表头。
    - 如果当前运行缺少某些列，自动填充空值。
    - 如果当前运行有多余列，**自动忽略**并发出警告（优先保证存储一致性）。
    - 强制使用 `extrasaction='ignore'` 确保写入不崩溃。

## 3. Notebook 物理属性防呆 (Anti-Mismatch)

- **问题**：用户可能在 CLI 中给 Semi-Auto 数据打上了 `Manual_Test` 的标签，导致分析结果混淆（这是你之前遇到的困惑）。
- **2.0 方案**：Notebook 加载数据时，会根据数据中是否包含预测点来计算 `condition`。如果物理检测到的类型（manual/semi）与用户在清单（Manifest）中定义的预期不符，直接打印红色 `❌ [LABEL MISMATCH]` 报错。
