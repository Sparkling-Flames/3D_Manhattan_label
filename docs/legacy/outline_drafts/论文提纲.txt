1研究问题
本研究聚焦于全景室内角点半自动标注的真实落地问题:当无法对所有样本进行逐一精标审核时,在控制偏差的前提下,量化半自动流程的效率收益,质量/可靠性变化,并进一步用多任务表现估计标注者的可靠度,从而实现任务分配与专家识别.
1.1研究问题(RQ)
RQ1:半自动初始化是否降低真实活跃标注时间？下降的比例是多少？
RQ2:在没办法全部都 人工精标 审核的情况下,半自动流程 是否能提升标注结果的可靠性和 一致性?并能 发现疏忽/异常编辑的反例?
RQ3: 能否在单任务,不同场景的表现估计标注者可靠度r_u和速率等结果,并利用擅长任务类型实现更优任务分配,从而提升总体质量与效率?

2.方法
量化改动幅度/工作量与质量/可靠性的指标体系,并明确各指标适用范围与局限性
其中:
改动幅度只说明改了多少,不能说明改的对不对
质量/可靠性独立统计
所有剔除/失败记录原因
	2.1标注对象与元信息
	标注输入以角点为主,不强制要求画墙面.每次提交需要同步三类信息,其中scope必填,其余字段在适用时填写
	scope(单选,决定是否进入主指标)
	In-scope:仅标注相机所在房间
Out-of-scope类:几何假设不成立/边界不可判定/错层多平面/证据不足

difficulty(可多选,解释耗时/误差来源):遮挡明显,拼接缝,玻璃/反光干扰,画质差/被mask影响,纹理弱,尽力调整但3d效果仍然不佳

model_issue(多选,仅半自动标注):预标注失效,配对异常,包含相邻空间,漏标,角点重复,角点错位
标注的规则
若门框/墙垛清晰:边界止于门框/墙垛处,不跨门洞,仍判定为in-scope
若无可靠停止点,需依赖猜测才行,选择OOS:边界不可判定,不强行构造立方体
2.2修改幅度/工作量(半自动标注组): 
任务t的模型初始化结果为P_t,标注者u的提交结果为A_{t,u}

2.1.1 图像空间Layout Polygon IoU (改动幅度)
用图像的polygon的交并比 展示从初始化到最终结果的形状的改动规模
IoU(X,Y)\ =\frac{\ |X\ \bigcap\ Y|\ }{|X\ \cup\ Y|}
其中X,Y为在全景图像空间上的layout投影多边形.目前标注者只用修改角点(如果需要画墙面也可以),X,Y有角点生成得到
当前代码将角点集合先构造成多边形后再计算IoU,其中角点多边形采用凸包(convex hul)以确保多边形有效.
凸包会将凹形结构外包为凸形,可能降低对跨门扩张/凹结构错误的敏感度,因此该IoU在论文中作为"改动幅度/一致性"的辅助证据,主结论优先依赖更几何一致的指标(如BoundaryRMSE与标准layout 2D/3D IoU).

在半自动初始化条件下,本指标默认比较模型初始化与标注者修订后的差异:
{IoU}_{edit}(t,\ u) = IoU(P_t,\ A_{t,u})

IoU越低通常表示改动越大,但它不保证改动是正确的,仅用作衡量工作量/改动幅度的方式之一

2.2.2边界RMSE
当角点数量不一致或者配对不稳定时,点多点RMSE易失真,所以采用角点生成的上/下边界曲线y_{ceil}\left(x\right) , y_{floor}(x)在离散网格上的误差作为几何改动幅度指标
BoundaryRMSE\left(X,Y\right)=\ 1Ni=1N[Δyceilxi2+ Δyfloorxi2 ]
2.2.3角点匹配RMSE(辅助指标)
该指标仅在角点配对成功且配对覆盖率足够高的情况下启用
预测角点的集合为{p_1,p_2,\ldots.,p_n},标注角点集合为{g_1,g_2,\ldots.,g_m},两点间欧式距离为
d(p,\ g)\ =[xp-xg2+ (yp-yg)2]
用匈牙利算法在两集合间求最小总代价的匹配π,匹配对上计算:
RMSE\ =\sqrt{(\frac{1}{N}\ \times\ {\sum_{i=1}^{n}{d(p_i},g_{\pi(i)})}^2}
其中\ N\ =\ min(n,\ m)  取预测角点,标注角点中数量较少的那个

2.3质量与可靠度
标注者间一致性(IAA)和标注者可靠度( r_u)关键原则:
	共识计算不包含被评估者自身,放置r_u系统性偏高
	 r_u只在纯人工标注集上进行估计,使其更接近人的能力,而不是工具把所有人拉到同一起点
2.3.1任务级一致性{\mathbit{IAA}}_\mathbit{t}
对任务t,收集所有标注者的提交结果A_{t,u}(需要任务标注者数量>=2),定义任务内一致性为两两IoU的中位数
\ \ \ \ \ \ \ {IAA}_t=\ {median}_{u\neq v}\ IOU(A_{t,u}\ ,\ A_{t,v})
   用中位数而非均值是想降低极端标注的影响.
\ \ \ \ \ \ {IAA}_t高意味着任务上存在稳定共识,用作专家识别 或者参考标准
2.3.2共识标注和去除自身后的共识(LOO)
在任务t上,定义共识代表为:
C_t= {argmax}_{A_{t,u}}{ {median}_{v\neq u}[IoU(A_{t,u},A_{t,v})]}
为了避免自我参与导致共识虚高,对每个标注者采用去除自身后的共识:
C_t^{\left(-u\right)} = {argmax}_{A_{t,w}\ ,\ \ \ w\in U_t\{u}}{ {median}_{v\neq w}[IoU(A_{t,w},A_{t,v})]}
并计算{IoU}_{LOO}(t, u) =IoU(A_{t,u},\ {C_t}^{(-u)}) 
直接与其他标注者比较:
{IoU}_{others-med}\left(t,\ u\right)={median}_{u\neq v}IoU(A_{t,u}\ ,\ A_{t,v})
2.3.3标注者可靠度\mathbit{r}_\mathbit{u}
标注者u在校准集合上的任务集合为T_u
是r_u = {median}_{t\in T_u}IoU(A_{t,u}\ {C_t}^{(-u)})
通过percentile bootstrap估计r_u的置信区间,仅当有效任务达到阈值时报告
2.3.4标准layout指标与门控
为了与HoHoNet / HorizonNet/Dula-Net 等前人研究的常用指标对齐,在满足门控条件scope = in-scope的样本上额外报告标准layout指标
2D IoU / 3D IoU:HoHoNet 风格的地面投影重合度(2D) 与考虑高度体积的重合度(3D)
由布局几何渲染出来的深度图(layout rendered depth RMSE /\delta):将layout 按几何关系渲染为全景视角下的每像素深度图,再对两份layout的深度图做像素级RMSE与阈值准确率
门控拆成两类进行展示:
可计算型门控:几何归一化失败,多边形无法构造,layout rendered RMSE生成失败等
可比型门控:点对点RMSE需要稳定匹配时

同时 n_pairs_mismatch 不再作为标准layout的门控条件,即不要求预测和标注角点对数一致,避免把 人工 显著纠错/高工作量的 样本排除
结构差异相关的信息(\Delta n_{pairs},是否存在奇数个数的点,配对失败等)用为分层/诊断的变量,用于报告和反例筛选

2.4结构差异信号
定义角点对数差:
\Delta n_{pairs}\left(t,u\right)=n_{pairs}\left(A_{t,u}\right)\ -{\ n}_{pairs}\left(P_t\right)
这个 是 难度/纠错的参考信号,用于分层报告和反例筛选
2.5 OOS(out-of-scope)统计
当任务被标注为OOS时,曼哈顿假设下的标准layout指标可能不具有意义
固定标注对象为相机所在的住房间的布局,当开放式连通导致

主报告汇总,标准layout指标对oos默认剔除,但单独报告oos数量及其分布

当scope选择任意OOS类时,主layout指标默认剔除,IAA与r_u的主估计默认剔除
单独报告 OOS比例,OOS类分布,OOS判定本身的一致性

当一个任务有些人选择了OOS 有些人选择 in-scope时,单独拎出来进行人工复核

3实验设置
实验目标为同时验证 半自动初始化的效率收益(RQ1),质量影响(RQ2),基于一致性/LOO可靠度的专家识别与任务分配收益(RQ3)
3.1实验目的
通过对照实验收集标注数据active time 与结果数据 ,在统一评估框架下输出指标,门控原因,反例案例库与统计置信区间
3.2参与者
参与者接受统一培训:
Label studio基础使用,工具使用,只标注相机所在主房间 ,OOS分流规则
Manual 与 semi两组中新手新手/有经验 按比例分配
任务随机抽样并按难度信号分层,使条件间可比
3.3条件分组与数据组织
3.3.1条件定义
实验组:提供模型初始化结果,标注者可修改或者保留
对照组:不提供初始化结果,标注者从零开始标注
	其余方面保持一致
	3.3.2六组数据划分
数据分组	规模	用途
Manual_Test	100	纯人工对照
SemiAuto_Test	100	半自动初始化对照
Calibration_manual	30(重复标注)	统一基准校准,估计r_u
Validation_manual	60	纯人工验证集
Validation_semi	60	半自动验证集
Calibration_semi	30(重复标注)	半自动条件下的一致性/反例对照(用于RQ2;不用于估计r_u)
Manual_Test 和 SemiAuto_Test 使用的是同一批图像,validation_manual和validation_semi使用的是同一批图像
3.4效率评估(RQ1)
核心指标:active time,用脚本捕捉用户实际进行标注的时长.
评估包含两个层级
Level1
对比Maunal 与Semi 条件下的active time 的分布,并按参与者经验分层展示
Level2
分析active time与改动幅度的关系, 确认 省时是否源于少改/瞎改
3.5可靠性,一致性与反例(RQ2/RQ3基础)
3.5.1校准集合\mathbit{C}_{\mathbit{manual}} 与\mathbit{C}_{\mathbit{semi}}(各30张,多标注)
Calibration_manual 对应校准集 C_{manual},每张图像至少有k>= 3 在manual 条件下独立标注,用于:
估计in-scope 样本的{IAA}_t分布
构建LOO公式并估计r_u, 仅manual,避免预标注把所有人拉到同一起点
构建稳定共识/不稳定任务的 反例候选
Calibration_semi 对应重复标注集 \mathbit{C}_{\mathbit{semi}},每张图像至少有k>= 3 在semi 条件下独立标注,用于:
估计semi 条件下的{IAA}_t / LOO分布,并与\mathbit{C}_{\mathbit{manual}}对比,使RQ2的一致性/可靠性是否提升 可检验
筛选半自动下异常编辑/跨门扩张等反例候选(与convex hull近似局限一起讨论)
3.5.2反例定义
反例候选通过可复现的规则筛选:
低一致性:{IAA}_t 低
修改模式异常: 改动幅度极大但一致性未提升,或者改动幅度极小但与他人显著不一致
门控失败:标准layout指标因门控不可用,但图像/其他信号差异显著
最终反例需要 人工进行复核确认
3.6专家识别与分配策略验证(RQ3)
3.6.1 统一基准条件估计\mathbit{r}_\mathbit{u}
为避免模型预标注的影响,r_u尽在calibration_manual数据集上估计,且in-scope
3.6.2验证集合V与任务分配对照
在validation_manual与validation_semi 内,分别对比两种任务分配策略:
1.分层随机分配,按难度/OOS指标
2.按r_u加权分配(难例优先分配给高r_u的标注者,或者根据不同标注者擅长哪种任务来分配)
一致性指标需要多标注来比较{IAA}_t:可在validation内划出子集做重复标注,或直接使用\mathbit{C}_{\mathbit{manual}} / \mathbit{C}_{\mathbit{semi}}作为一致性对照集
3.6.3验证指标与展示
核心验证指标:
一致性:验证子集上的{IAA}_t是否提升
效率:相同预算或者产出下active time是否下降
鲁棒性:OOS比例与子类分布是否更稳定,门控失败率是否下降
展示内容
r_u与active time 的关系
分配策略调整前后,一致性/效率/反例 的变化

5.标注数据重训练
5.1目标
利用积累的人工修改的标注数据改进初始化模型,目标是
提高初始化质量:{IoU}_{edit}(t,\ u) 
		降低结构性错误:降低配对异常,预标注失效,识别角点错误,跨门扩张等问题
		降低门控失败率:提升可计算性
5.2数据分组
分为train/valid/test 
没有mask 的图像依照 HoHoNet要求进行mask,提升模型精确度
5.3评估方式
在同一套系统下评估旧初始化 vs 新初始化 对标注过程的影响
6.可能出现的问题
学习效应导致 数据不准确
