#### A1. `create_labelstudio_split.py` ── 重构为阶段感知版本

**现状**：只有 `split_tasks(main, calib, val, gold)` 四分区，无阶段概念。  
**目标**：增加 `--stage {pilot, prescreen, calibration, main}` 参数，不同阶段输出不同分区。

**开发要点**：

1. **Pilot 阶段**（`--stage pilot`）
   - 输出 `pilot.json`，固定 $N_{img}=15$，随机种子写入清单。
   - 不与其他阶段图像池重叠（图像集从全局 pool 中扣除）。

2. **PreScreen 阶段**（`--stage prescreen`）
   - 输出 `prescreen_manual.json`（$N_{img}=30$，含 $n_{anchor}=12$ 张专家参考标记字段 `is_anchor=true`）和 `prescreen_semi.json`（$N_{img}=30$，不与 manual 图像重叠）。
   - `prescreen_semi.json` 中每个任务需带 `init_type` 字段（`clean` / `misleading`），由扰动清单注入（见 A2）。
   - 专家参考 Trap 样本需在 JSON 中标记 `has_expert_ref=true`。

3. **Calibration 阶段**（`--stage calibration`）
   - 输出三个子集 JSON：`calibration_anchor.json`、`calibration_core.json`、`calibration_reserve.json`。
   - 保证三个子集图像不重叠；`anchor` 所有通过 PreScreen 的 worker 全员覆盖。
   - 支持 `--hard_ratio` 参数（默认 0.4）：在 calibration_core 中强制保留该比例的"高 $g_t$ 触发"样本（用形态学规则预标记）。
   - 输出 `split_manifest.json`：记录随机种子、阶段、图像集合与子集划分，入库版本控制。

4. **Main 阶段**（`--stage main`）
   - 输出 `manual_test.json`（$N_{img}=100$，$k=1$）、`semiauto_test.json`（$N_{img}=100$，$k=1$）、`validation_semi.json`（$N_{img}=60$，初始 $k_0=3$）、`gold_manual.json`（$N_{img}=20$，$k=3$）。
   - 保证所有 Main 图像不与 PreScreen/Calibration 图像重叠（从 pool 中物理扣除）。

**验收标准**：`python tools/create_labelstudio_split.py --stage calibration --seed 42 --dry-run` 输出清单无重叠断言通过。

#### A2. 扰动算子库（PreScreen Semi 的误导性初始化）

**现状**：只有 `clamp_occ_corners.py`（单一算子，无清单）。  
**目标**：新建 `tools/generate_misleading_init.py`，实现提纲附录 A1 描述的算子库。

**开发要点**：

1. 实现以下 **4 类算子**（以 Pilot 中高频 model_issue 为依据，以提纲附录 A1 为规范）：

   | 算子名 | 对应 model_issue | 实现逻辑 |
   |---|---|---|
   | `overextend_adjacent` | overextend_adjacent | 随机在 x 方向延伸最左或最右 corner pair，幅度 `λ ∈ [0.1, 0.3]` × width |
   | `corner_drift` | corner_drift | 对全部 corner keypoint 加 2D 高斯噪声，`σ = λ × height`，`λ ∈ [0.01, 0.05]` |
   | `corner_duplicate` | corner_duplicate | 随机复制 1-2 个 corner pair，x 偏移 5-15 px |
   | `topology_collapse` | topology_failure / fail（小配额） | 打乱 corner 配对顺序，使天花板点出现在地板位置 |

2. 每个算子接受 `(corners_px, operator, lambda, seed)` → 返回 `perturbed_corners_px`。

3. 生成函数 `generate_misleading_batch(image_ids, operator_dist, seed)` → 输出包含 `{image_id, operator, lambda, seed, perturbed_corners}` 的清单 JSON，**写入后不得再修改**（只读锁）。

4. 验收检查：给定同一 `seed`，两次运行输出完全一致（可复现性断言）。

**注意**：不需要可视化界面，只需 CLI 脚本生成 JSON 清单。

#### B1. `analyze_quality.py` ── LOO 共识与 IAA 扩展

**现状**：`compute_consistency` 只计算 pairwise IoU 均值（不是中位数、不是 LOO）。  
**目标**：在原函数基础上**增加**（不破坏旧接口）以下函数。

**开发要点**：

1. **`compute_iaa(annotations) → float | None`**
   - 计算 $\mathrm{IAA}_t = \mathrm{median}_{u \neq v} \mathrm{IoU}(A_{t,u}, A_{t,v})$。
   - 需要 $k \geq 2$；只使用 in-scope 的标注（复用已有 `parse_quality_flags_v2` 过滤）。
   - 使用 `Polygon(corners).convex_hull` 作为 in-scope 的多边形代理（与 `compute_consistency` 逻辑一致）。
   - 返回 `(iaa: float | None, n_pairs: int, gated: bool, gate_reason: str)`。

2. **`compute_loo_consensus(annotations) → dict`**
   - 对每个标注者 $u$，计算 $C_t^{(-u)}$（在 $U_t \setminus \{u\}$ 上寻找中位数 IoU 最大的工人）。
   - 返回 `{uid: {"loo_consensus_uid": str, "iou_loo": float | None, "iou_others_med": float | None, "gate_reason": str}}`。
   - 特别处理 $k=3$ 退化：当 $|U_t \setminus \{u\}| = 2$，用 `iou_others_med` 作为替代，并设 `k3_degenerate=True`。

3. **`compute_iou_edit(prediction_corners, annotation_corners) → float`**
   - $\mathrm{IoU}_\mathrm{edit}(t,u) = \mathrm{IoU}(P_t, A_{t,u})$，其中 $P_t$ 来自 task 的 prediction 字段。
   - 复用 `compute_iou`，输入为 convex hull polygon points。

**接口要求**：上述函数须可从 `main()` 的 per-task 循环中调用，结果列追加到输出 CSV 的每一行（per-annotation 行含 `iou_loo`, `iou_others_med`, `iaa_t`, `k3_degenerate`）。

#### B2. `tools/compute_reliability.py` ── 新建，$r_u$ / $r_u^{(s)}$ 估计器

**现状**：不存在。  
**目标**：新建独立脚本，输入为 `analyze_quality.py` 生成的 CSV（含 `iou_loo` 列），输出 worker 级别可靠度报告。

**开发要点**：

1. **全局可靠度 $r_u$（主估计）**
   - 输入筛选：`dataset_group == "calibration_manual"`，`is_oos == False`，`task_k >= 4（冗余下限）`。
   - 统计口径：per worker，取 `iou_loo` 的 **中位数**。
   - BCa Bootstrap：调用 `_bootstrap_ci(iou_loo_values, np.median, n_iters=10000, ci=0.95)`（复用已有函数），输出 `r_u, lcb_r_u, ucb_r_u, h_u`。
   - 停机判断：若 `h_u <= ε_r`（参数，默认 0.05），标记 `precision_ok=True`；否则 `False`。
   - 若 `n_tasks_used < N_r_min`（默认 15），标记 `low_precision=True`（优先于 CI）。

2. **场景特异可靠度 $r_u^{(s)}$**
   - 输入：`scene_label` 列（来自多选 tag 的共识标签，见 B3），筛选 Calibration_manual same as above。
   - 对每个 `(worker_id, scene)` 组合：若 `N_{u,s} >= N_us_min`（备选 5 或 6，冻结后固定），计算中位数 + bootstrap CI。
   - 输出字段：`r_u_s, lcb_r_u_s, ucb_r_u_s, n_us, scene, activated`（若 `N_{u,s} < N_us_min` 则 `activated=False`，路由退化到 `r_u`）。
   - 同时输出 `activation_rate`：各场景中 `activated=True` 的 worker 比例，用于 Discussion 中的退化率披露。

3. **停机逻辑报告**
   - 输出 `calibration_stoppage_report.csv`：per worker，含 `current_n_cal`, `h_u`, `precision_ok`, `reserve_needed`（是否需要从 reserve 追加），`low_precision`。

4. **$r_u^{(0)}$ 估计（PreScreen 阶段锚点）**
   - 输入筛选：`dataset_group == "prescreen_manual"`，`has_expert_ref == True`。
   - 计算 $r_u^{(0)} = \mathrm{median}_{t} \mathrm{IoU}(A_{t,u}, G_t)$（$G_t$ 来自专家参考字段 `expert_ref_corners`）。  
   - 输出放入 `prescreen_gate_report.csv`，供 A 和 C 使用。

5. **Spammer score $S_u$**
   - 5 个分量（$p_{fail}, p_{NA}, p_{idle}, p_{trust}, p_{var}$）均按提纲公式实现，阈值以 `config` dict 传入（不硬编码，Pre-registration 锁定时写入配置 JSON）。
   - 输出 `worker_profile.csv`：含 `worker_id, r_u, lcb_r_u, S_u, group（Stable/Vulnerable/Noise）`，供报告模块使用。

**验收标准**：在 `tests/` 目录下补充单元测试，覆盖：①单 worker 全相同标注时 $r_u=1.0$；② $k=3$ 退化时函数不崩溃；③ bootstrap n_iters 参数有效影响 CI 宽度。

#### B3. `analyze_quality.py` ── per-tag 共识与 Fleiss' Kappa 扩展

**现状**：`parse_quality_flags_v2` 只返回几个 bool flag，没有 per-tag 投票率和 Kappa。  
**目标**：在 per-task 聚合层新增共识计算。

**开发要点**：

1. **`compute_tag_consensus(annotations, tag_field) → dict`**
   - `tag_field` ∈ `{"difficulty", "model_issue"}`。
   - 对每个 tag $k$：统计 $n_{selected}$, $n_{total}$（NA 单独统计），返回 `c_k, conf_k, disagree_k`。
   - 任务级 `disagree_t = 1 - min(conf_k for k in K_t)`（保守代理）。
   - NA 缺失率：`na_rate_k = sum(NA) / total_annotations`，写入输出。

2. **`compute_fleiss_kappa(annotation_matrix) → float`**
   - 输入：`(n_tasks × n_annotators × n_tags)` 形状的 bool 矩阵（或 per-tag 的独立矩阵）。
   - 对每个 tag 独立计算 Fleiss' $\kappa$，报告 `median κ`。
   - 使用 `statsmodels.stats.inter_rater.fleiss_kappa` 或手动实现（确认依赖可用）。
   - 当 tag 频率 < 5% 时标记 `low_freq=True`，输出 `kappa_warning`。

3. **输出追加**：CSV 新增列 `consensus_tags`（逗号分隔的共识 tag 列表）、`disagree_t`、per-tag `c_k_*` 列。

#### B4. `analyze_quality.py` ── $g_t$ 结构诊断代理

**现状**：无。  
**目标**：新增 `compute_g_t(prediction_corners) → bool`，在主循环中对 prediction 字段计算。

**开发要点**：

1. **触发条件**（复现提纲 §结构差异信号）：

#### C1. `aggregate_analysis.py` ── T/I/M 口径筛选器扩展

**现状**：`compute_summary_stats` 直接对全量 rows 分组，没有口径概念。  
**目标**：在 `merge_csv_files` 下游增加 `filter_by_scope(rows, scope) → rows`。

**开发要点**：

1. **实现三个筛选函数**：
   - `filter_T(rows)` → 全量（仅去除物理重复行），报告 OOS 比例与门控失败原因分布。

#### C2. `tools/run_prescreen_gate.py` ── 新建，PreScreen 门控脚本

**现状**：不存在。  
**目标**：新建 MVP 级别脚本，读取 PreScreen 质量报告 CSV，输出哪些 worker 通过/不通过门槛。

**开发要点**：

1. **输入**：`prescreen_manual_quality.csv`（由 `analyze_quality.py` 生成），`worker_profile.csv`（由 B2 生成）。

2. **门控条件**（所有阈值从 `prescreen_config.json` 读入，不硬编码）：
   - `gate_fail_rate <= τ_fail`：可计算型门控失败率。

3. **输出**：
   - `prescreen_gate_report.csv`：per worker，含每条门控的 pass/fail 明细与最终 `admitted`（布尔）。
   - 终端打印摘要（通过人数、失败人数、各门控触发次数）。

4. **设计原则**：纯规则引擎，无任何图形输出，脚本在 2 秒内跑完（不做 bootstrap，bootstrap 已在 B2 中预计算）。
