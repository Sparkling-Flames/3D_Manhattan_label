# 三人开发实施方案（严格对齐论文提纲）

版本：v1.0（2026-02-20）  
适用范围：HOHONET 标注系统与分析工具链  
对齐来源：`sections/01~08` 全部章节 + 导师语音第7点（三段式 protocol、先把实验跑起来、软件逐步完善）

---

## 1. 目标与约束（与提纲一致）

### 1.1 目标（不可变）
1. 按 `Pilot -> PreScreen -> Main` 三段式协议完成可审计实验闭环。  
2. 形成 RQ1/RQ2/RQ2b/RQ3 对应的数据、指标、统计与过程性证据。  
3. 输出论文第4章要求的可审计产物：三级口径 T/I/M、Worker×Scene 矩阵、反例库、过程归因链。  
4. 在 non-IID/OOD 条件下完成离线路由策略对比（Random/Global/Full）并给出预算-质量折衷证据。

### 1.2 约束（来自提纲与导师第7点）
- 允许“流程先手动跑通 + 软件后续补自动化”，但必须保留完整日志与阶段隔离证据。  
- 不允许事后调参：MDE、阈值、映射规则、算子库、随机种子需冻结并留档。  
- 不允许静默过滤：所有样本进入 T 口径披露。  
- 研究假设以受控组织环境为前提，恶意标注者不作为主流程前提。

说明：以下判定基于当前 `tools/` 脚本清单与代码能力核查。

| 提纲需求 | 现有工具 | 覆盖判定 | 缺口等级 | 结论 |
|---|---|---|---|---|
| 三段式数据切分（Pilot/PreScreen/Calibration/Main） | `create_labelstudio_split.py` | 部分覆盖 | P0 | 仅支持 main/calibration/validation/gold 随机切分，未显式建模三段式状态与冻结协议 |
| active_time 采集与汇总 | `cors_server.py`, `analyze_quality.py`, `lead_time_stats.py` | 部分覆盖 | P1 | 采集可用，但审计字段与(任务,标注者)粒度一致性仍需强化 |
| RQ1 效率统计（中位数差、HL、CI） | `analyze_quality.py`, `aggregate_analysis.py` | 部分覆盖 | P1 | 有基础统计，缺少提纲要求的完整推断流程（Mann-Whitney/HL/BCa 一体化输出） |
| RQ2 一致性/可靠性（IAA、LOO、r_u） | `analyze_quality.py` | 覆盖较好 | P1 | 核心已具备，但尚未形成“章节级可审计报表模板” |
| RQ2b 加权共识（w_u、w_max 锁定） | 无独立脚本 | 未覆盖 | P0 | 缺少加权共识引擎、w_max 选择协议实现、敏感性网格输出 |
| 多选标签共识（per-tag、disagree、Fleiss’ Kappa） | 无独立脚本（主脚本未形成完整模块化输出） | 部分覆盖 | P0 | 需补全 per-tag 共识计算与统计输出 |
| Worker 画像（LCB(r_u), S_u）+ 3组分类 | 无独立脚本 | 未覆盖 | P0 | 需新增 worker characterization 工具 |
| Worker×Scene 矩阵 r_u^(s) ±CI + 局部失效 | 无独立脚本 | 未覆盖 | P0 | 需新增场景可靠度估计与矩阵导出工具 |
| RQ3 路由离线模拟（Random/Global/Full，k0-kmax） | 无 `routing/simulate` 工具 | 未覆盖 | P0 | 关键缺口，必须新增 |
| non-IID/OOD stress split（基于 d_t/g_t） | 无独立脚本 | 未覆盖 | P0 | 需新增 OOD 切分与应激集构造工具 |
| 反例库自动产出（规则筛选+可视化+复核清单） | `save_quality_figures.py` 部分可视化 | 部分覆盖 | P1 | 需补“规则筛选、IID/OOD对比、复核清单”一体化导出 |
| 三级口径 T/I/M 报告包 | `analyze_quality.py` 有部分字段 | 部分覆盖 | P1 | 需固定模板化输出（表A/B/B2/C/D） |
| 预注册与冻结证据（seed/阈值/算子） | `create_labelstudio_split.py` 有报告 | 部分覆盖 | P1 | 需扩展为统一 manifest + 阶段锁定审计 |

缺口等级定义：P0=不开工主实验会阻塞；P1=可跑但不可审稿级复现；P2=优化项。

## 3. 缺失工具清单（必须新增，按优先级）

## 3.1 P0 必须新增（先做）

1) `tools/simulate_routing.py`  
- 目标：实现 Random / Global / Full 三策略离线模拟。  
- 输入：任务-标注者多标注数据、`r_u`、`r_u^(s)`、`d_t`、`g_t`、预算参数(k0,kmax)。  
- 输出：每策略的一致性代理、失败率、active_time、k_used 分布、预算效率。  
- 验收：可复现（固定 seed）、可回放（逐任务分配日志）。

2) `tools/compute_weighted_consensus.py`  
- 目标：实现 per-tag 加权共识，支持 `w_u` 映射与 `w_max` 预注册锁定。  
- 输出：未加权/加权对比、Δacc/Δkappa、Bootstrap CI、敏感性网格。  
- 验收：主评估与训练估计严格隔离（fold 内无泄漏）。

3) `tools/build_ood_stress_split.py`  
- 目标：基于标注前代理 `d_t`、`g_t` 构造 IID 与 non-IID/OOD 切分。  
- 输出：切分清单、占比报告、重叠率报告(H vs Validation_OOD)。  
- 验收：切分规则可复现，且与后验标签解耦。

4) `tools/worker_characterization.py`  
- 目标：计算 `LCB(r_u)`、`S_u`、3类功能组，输出画像散点数据与分组表。  
- 输出：worker_profile.csv、group_assignment.csv、plot_data.json。  
- 验收：分量定义固定（p_fail/p_NA/p_idle/p_trust/p_var），阈值可配置并冻结。

5) `tools/scene_reliability_matrix.py`  
- 目标：估计 `r_u^(s)` ±95%CI，生成 Worker×Scene 矩阵与局部失效标记。  
- 输出：worker_scene_matrix.csv、local_failure_flags.csv。  
- 验收：低样本格子按规则标 `--`，并输出退化比例。

## 3.2 P1 重要新增（主实验前补齐）

6) `tools/run_protocol.py`（流程编排器）  
- 目标：把 Pilot/PreScreen/Main 的步骤、输入、输出、冻结点串成可执行流水。  
- 输出：每阶段 manifest、日志、冻结快照。  
- 价值：落实导师第7点“两条腿走路”，支持手动步骤但可追溯。

7) `tools/generate_audit_pack.py`  
- 目标：一键导出第4章审计材料（T/I/M、表A/B/B2/C/D、反例清单）。  
- 输出：paper-ready CSV/MD/图表目录。  
- 验收：结果与提纲字段一一对应。

8) `tools/export_counterexample_bank.py`  
- 目标：按固定规则筛选反例，导出可视化叠加与专家复核模板。  
- 输出：counterexamples.csv、review_sheet.xlsx、可视化目录。

## 4. 现有工具改进清单（客观可执行）

## 4.1 `tools/create_labelstudio_split.py`（当前：部分满足）

**问题**
- 目前按总量随机切分，缺少“阶段化池管理 + 冻结字段完整性”。

**改进**
1. 增加 stage-aware 参数：`--pilot-count --prescreen-manual --prescreen-semi --calib-core --calib-reserve --main-test --main-validation --gold`。  
2. 增加硬约束检查：跨阶段图像不重叠；Trap/Gold 与 Main 物理隔离。  
3. 增加冻结清单字段：`seed`、算子版本、配置 hash、时间戳、生成脚本版本。  
4. 产出 split manifest（JSON）可直接附录引用。

## 4.2 `tools/analyze_quality.py`（当前：核心强，但需模块化补齐）

**问题**
- 功能集中在单脚本，尚未完整覆盖 RQ2b、Scene 矩阵、画像分型、路由输入中间件。

**改进**
1. 抽离共识模块：支持 per-tag 共识、disagree、kappa 输出。  
2. 增加“路由输入导出”：任务级风险特征、worker级可靠度特征标准格式。  
3. 增强审计输出：T/I/M 口径统计模板化，便于论文复现。  
4. 增加统计接口：输出中位数差、HL、BCa CI 所需原始分层数据。

## 4.3 `tools/aggregate_analysis.py`（当前：聚合基础可用）

**问题**
- 以均值聚合为主，缺少提纲定义的主统计流程与效应量门槛判据。

**改进**
1. 增加非参数检验与效应量：Mann-Whitney、KS、McNemar、Cohen’s d、HL。  
2. 增加 BCa bootstrap CI 与 MDE 判定模块。  
3. 增加多重检验校正（Bonferroni）。

## 4.4 `tools/lead_time_stats.py`（当前：可做质检，口径需统一）

**问题**
- 目前对任务级 active_time 采用“跨标注者取max”近似，不满足某些分析需 `(task,annotator)` 粒度。

**改进**
1. 新增输出粒度选项：`--granularity task|task_annotator|session`。  
2. 默认输出与 `analyze_quality.py` 完全同口径的校验报告。  
3. 增加异常日志审计项（unknown id、缺失script_version、多session比例）。

## 4.5 `tools/save_quality_figures.py`（当前：图表生成可用）

**问题**
- 还不能直接生成“过程证据链”所需图组（功能组对比、IID vs OOD 反例频率变化）。

**改进**
1. 新增图组：功能组对比、场景分层失败率、OOD 子集对比。  
2. 输出固定命名规范，直接用于 Overleaf 引图。

## 5. 三人分工（RACI）

角色定义：
- 成员A：协议与实验负责人（Protocol Owner）  
- 成员B：工程与工具负责人（Toolchain Owner）  
- 成员C：统计与审计负责人（Analysis Owner）

| 工作包 | A | B | C | 说明 |
|---|---|---|---|---|
| 协议冻结（阈值/MDE/seed/算子） | R | C | C | A最终签字 |
| 数据切分与任务包生成 | A | R | C | B实现，A验收隔离性 |
| PreScreen执行与准入清单 | R | C | A | C参与阈值核验 |
| active_time与日志链路 | A | R | C | B保稳定采集 |
| 加权共识实现与评估 | C | C | R | C主导统计正确性 |
| Worker画像与分组 | C | C | R | 输出论文图D数据 |
| Worker×Scene矩阵 | C | C | R | 输出表C |
| 路由模拟器 | A | R | C | B实现，A定义策略规则 |
| 反例库与过程证据 | A | C | R | C主导筛选规则 |
| 论文结果包导出 | A | C | R | C汇总，A审稿口径把关 |

R=负责执行；A=最终负责；C=协作。

## 9. 验收标准（DoD）

## 9.1 工程DoD
1. 所有新工具支持命令行与 `--help`。  
2. 每个工具都有输入样例、输出样例、异常退出码。  
3. 所有阶段产物可追溯到 manifest（含 seed、版本、时间戳）。

## 9.2 方法DoD（与提纲对应）
1. RQ1：输出中位数差、HL、95%CI、MDE判定。  
2. RQ2：输出 IAA 分布差异、反例类型频率、可靠性报告。  
3. RQ2b：输出加权/未加权对照与独立参考一致性差。  
4. RQ3：输出 Random/Global/Full 在 IID 与 OOD 子集下对比。  
5. 报告章节产物完整：T/I/M、表A/B/B2/C、图D、反例库。

## 9.3 审稿防御DoD
1. 可回答“阈值是否事后调参”：有冻结证据。  
2. 可回答“是否选择性过滤样本”：有 T 口径全量披露。  
3. 可回答“离线路由局限”：有边界声明与可复现实验日志。

## 10. 风险与应对（执行层）

| 风险 | 触发信号 | 影响 | 应对 |
|---|---|---|---|
| 标注资源不足 | 周完成任务数低于计划70% | 工期延迟 | 先保证 PreScreen 与 Calibration，Main 采用最小可发表规模 |

| OOD切分不稳定 | H集<30 或重叠异常 | RQ3统计无效 | 放宽到预注册敏感性区间并固定一次 |
| 统计口径不一致 | 不同脚本结果不一致 | 论文风险高 | 建立“单一事实源”检查（以 analyze_quality 输出为准） |
| 画像分桶稀疏 | 某类样本数过低 | 结论脆弱 | 主文坚持3类功能组，细分仅放附录 |
