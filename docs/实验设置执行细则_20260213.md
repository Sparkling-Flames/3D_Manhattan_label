# 实验设置执行细则（从提纲3.1抽离）

日期：2026-02-13
适用范围：论文提纲3.1对应的实验实施与审计留痕

## 1. 候选池与元信息

- 候选池：`mp3d_layout/test/img` 共 458 张，冻结为 `manifest.csv`。
- 客观剔除：仅允许文件损坏、不可加载、分辨率异常、任务创建失败；全部在口径 T 单列披露。
- 预先计算元信息：
  - 冻结特征 `e_t`（默认 HoHoNet 共享 pre-head latent，经宽度池化后得到图像级向量）。
  - 结构风险代理 `g_t`（多边形构造失败、拓扑崩溃、角点重复/奇数、配对失败）。
  - OOD 风险分数 `d_t`（Calibration 确定后，用归一化特征到参考库的第 K 近邻距离，K=10），并据此计算触发器 `I_t^{OOD}`。

## 2. 抽样总原则

- 分阶段互斥：Pilot、PreScreen、Calibration、Test、Validation 图像不重叠。
- Trap 物理隔离：Trap 与 Main 所有集合隔离。
- 先分层后随机：按预注册配额抽样，禁止事后挑样。
- OOD 只用于分层与触发，不用于静默过滤。

## 3. 阶段与配额

- Stage 0 Pilot：`N_img=15, k=2`。
- Stage 1 PreScreen：
  - `PreScreen_manual = 30`，其中 `20-22` 张为专家参考锚点，其余 `8-10` 张为随机 non-anchor。
  - `PreScreen_semi ~= 18`，包含正常初始化对照子集与误导性初始化 trap 子集。
- Stage 2 Calibration：
  - `Calibration_manual = 100, k=4`。
  - 内部分为：`Calibration_anchor = 12`、`Calibration_core = 75`、`Calibration_reserve = 13`。
  - 其中 `Calibration_anchor` 是全员公共比较子集，不是新的专家真值锚点。
  - `Calibration_semi = 25, k=4`，仅用于 RQ2 条件对照，不参与 `r_u` 或 `r_u^(s)` 估计。
- Stage 3 Main：
  - RQ1 Test：Manual_Test 与 SemiAuto_Test 同图，`N_img=100, k=1`。
  - RQ3 Validation：Validation_semi `N_img=60, k_0=3, k_max=5`；Gold_manual `N_img=20, k=3`。

## 4. non-IID/OOD 应激与补抽样

- Hard 子集：从 Validation 构造 `H`，要求 `|H| >= 30`。
- 入选依据：`I_t^{OOD}=1` 或高 `g_t`。
- 标注后验证：`S_hard` 占比 `>= 70%`。
- 若未达标：按同一分层规则从缓冲池补抽样，并记录补抽样次数、原因与版本。

## 5. 隔离与日志

- Trap 不进入 Calibration 的 `r_u`/`r_u^{(s)}` 估计，不进入 Validation/Test 策略评估。
- Main 中 Test/Validation 图像与 Calibration 图像不重叠。
- 每阶段保留 task_id 清单、导出文件版本、时间戳、参与工人列表、纳入/剔除规则。

## 6. 数据版本与评测口径

- `test` 与 `test_no_occ` 图像集合一致，仅 `label_cor` 不同。
- benchmark 主报告：`test/label_cor`。
- 敏感性分析：`test_no_occ/label_cor`。
- 半自动预标注输入来自 `test/img` 推理输出，与 `label_cor` 版本无关。
