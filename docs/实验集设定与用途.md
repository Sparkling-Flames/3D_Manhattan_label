# 实验集设定与用途

本文当前采用三类前置样本池：manual 几何锚点池、semi 纠错/盲信池、OOS gate 池。三者角色不同，不应混用。

## 1. PreScreen_manual

### 1.1 manual 锚点集
- 目标规模：20 到 22 个。
- 所属总池：
	- 当前设想为 PreScreen_manual 总共 30 个，其中 20 到 22 个为人工锚点，剩余 8 到 10 个为非锚点随机样本。
- 选择原则：
	- 人工确保覆盖关键场景和关键难度层。
	- 只纳入可稳定给出几何专家参考的 In-scope 样本，不把 OOS/证据不足样本混入几何 GT 锚点。
	- 目标是让 prescreen 真能看到不同 failure family，而不是全是容易样本。
- 当前优先覆盖的 manual 家族：
	- 非常简单：作为低难度基准。
	- 遮挡明显但仍可闭合：检验在遮挡下的基本稳定性。
	- 拼接缝及拉伸：检验 seam/stretch 诱导的局部折点错误。
	- 玻璃/反光：仅在专家能稳定给出参考时纳入；高歧义样本只保留为待复核或审计样本，不强行作为硬锚点。

### 1.2 manual 非锚点集
- 规模：8 到 10 个。
- 采样方式：从定义好的候选池里随机抽取。
- 目的：保留真实 base-rate，避免整个 prescreen 变成人工拼装的 showcase。
- 作用：
	- 让 admission screening 不只建立在“精心挑选的典型题”上。
	- 用于观察工人在较自然分布上的稳定性，但不承担主要阈值拟合职责。

## 2. PreScreen_semi

### 2.1 semi 锚点集
- 目标规模：18 个左右。
- 角色：
	- 不承担 manual 几何基准作用。
	- 主要用于测量纠错能力、盲信初始化风险、以及典型失败家族上的修正能力。
- 当前推荐结构：
	- 6 个“模型标注质量好”的正常初始化对照样本。
	- 12 个误导性初始化样本，覆盖 4 个左右失败家族，每类约 3 个；若某类自然失败不足，则由规则化扰动补足。
- 当前 semi 自然失败家族优先级：
	- 过度解析
	- 角点错位
	- 角点重复
	- 跨门扩张
	- 模型预标注失败
- 备注：
	- 当前 trap bank 中“漏标”“拓扑崩溃”尚未形成稳定自然案例，不应被设为 prescreen 必须覆盖家族；若后续补齐，可作为扩展家族加入。

## 3. OOS gate 池

### 3.1 用途
- OOS gate 池不进入 manual 几何 GT 锚点。
- 其职责是检验工人是否能正确拒绝继续几何标注，或正确识别“当前证据不足/几何假设不成立”。

### 3.2 当前优先覆盖的 OOS 家族
- 边界不可判定
- 证据不足
- 错层/天花板下凸
- 几何假设不成立（如弧形墙）

### 3.3 使用原则
- OOS gate 的判断正确性应与 manual 几何正确性分开计分。
- 它是 scope/gate 能力测试，不是几何精标能力测试。
- 因此不能把 OOS gate 样本混写成 manual 锚点数量的一部分。

## 4. 当前最适合的总结构

- PreScreen_manual：30 个
	- 20 到 22 个 manual 几何锚点
	- 8 到 10 个随机非锚点
- PreScreen_semi：18 个左右
	- 正常初始化对照 + 误导性初始化 trap
- OOS gate：独立小池
	- 不与 manual 几何锚点混池

## 5. 这样设计的原因

- manual 锚点负责测“基本人工几何能力”。
- semi 锚点负责测“面对模型初始化时是否会盲信、是否会纠错”。
- OOS gate 负责测“是否能拒绝不该继续做几何闭合的样本”。

如果把三者混成一个 prescreen 集合，审稿时最容易出现的问题是：
- 你到底在测工人的几何能力，还是在测其纠错能力？
- 你到底在测 In-scope 标注质量，还是在测 OOS 拒绝能力？
- 你最后得到的 admission threshold 具体对应哪一种 estimand？

因此，当前版本应坚持三池分离、角色分离、计分分离。
