# 顶刊审稿视角：图表清单与分层策略（基于提纲 v2）

## 0. 总评（理性严苛）

从顶刊审稿标准看，你的提纲优势是“审计链完整”，风险是“图多但主线分散”。
若要提升中稿概率，图表必须服从一个最小闭环：

1. 先证明数据透明（T/I/M 不隐式丢样本）
2. 再证明核心收益（效率 + 可靠性）
3. 再证明收益来源（分层/工人/场景）
4. 最后给可追溯反例与过程证据（不是故事图）

## 1. 论文正文必备图（Paper Core，建议 6 图 + 3 表）

### Fig.P0（必备）T/I/M Coverage Funnel + Missingness

- 对应提纲：4.1
- 回答质疑：是否存在选择性汇报（selection bias）
- 最低要求：显示 T→I→M 的样本/任务数；同时给 gate reason 占比

### Fig.P1（必备）Active Time：Manual vs Semi

- 对应提纲：RQ1、4.2
- 回答质疑：效率改进是否真实
- 最低要求：中位数差 + 95% CI + 分布图（箱线/小提琴）

### Fig.P2（必备）Reliability 分布对比

- 对应提纲：RQ2、2.4
- 回答质疑：半自动是否伤害一致性
- 最低要求：Calibration_manual vs Calibration_semi 的 IAA 或 IoU_LOO 分布

### Fig.P3（建议）路由策略主结果图（IID vs non-IID）

- 对应提纲：RQ3、3.0、2.8
- 回答质疑：路由收益是否只在特定分布成立
- 最低要求：Random / Global / Stratified 三策略在一致性代理与失败率上的对比

### Fig.P4（建议）工人画像二维散点图（摘要版）

- 对应提纲：4.2、4.4
- 回答质疑：收益来源是否可解释
- 最低要求：以 $(S_u,\ \mathrm{LCB}(r_u))$ 为坐标的二维画像散点图（用于展示三组 Stable/Vulnerable/Noise）；颜色=三组；形状=Vulnerable 触发原因；标注每人样本量与不确定性（CI/点大小等）

### Fig.P5（必备）反例案例拼图（规则筛选）

- 对应提纲：4.3、4.4
- 回答质疑：异常是否可复现、可归因
- 最低要求：每类反例至少 1 例，附规则触发字段，不可只给视觉截图

### Table.A（必备）Δn_pairs 分层表

- 对应提纲：4.2
- 字段：覆盖率、门控失败率、active_time、IoU/BoundaryRMSE

### Table.B（必备）difficulty/model_issue 分层表

- 对应提纲：4.2
- 字段：频率、一致性代理、Fleiss’ kappa（或替代一致性指标）

### Table.A1（必备）反例清单

- 对应提纲：4.3
- 字段：task_id、type、触发规则、复核结论、责任链字段

## 4. 审稿员最可能追问（你需用图直接回答）

1. 你是不是只报告“能算出来”的样本？
   - 用 Fig.P0 + Table.P0 回答。
2. 半自动快了，但是不是质量下降了？
   - 用 Fig.P1 + Fig.P2 联合回答。
3. 路由收益是不是偶然/过拟合某一组？
   - 用 Fig.P3（IID/non-IID 双情景）回答。
4. 反例是不是挑选展示？
   - 用 Table.A1（规则触发+复核字段）回答。

## 5. 出图执行原则（强约束）

- 正文只放“闭环最小图集”，其余全进附录/内部。
- 每张图必须绑定“回答哪条质疑”。
- 每张图要有可追溯表格源（CSV），并与口径 T/I/M 对齐。
- 不允许“只给图不给分母”；必须显示样本数与缺失率。
- 所有阈值（例如 n_min=2）在图注或表注固定披露。

## 6. 与当前 notebook 的分工落地

- v1（internal）：保留全量诊断，输出 `internal_figures_v1/`
- v2（paper slim）：仅保留正文最小闭环 + 必要附录表，输出 `paper_figures_v2/`

建议流程：

1. 先跑 v1 排查数据质量与门控稳定性；
2. 锁定图集后再跑 v2 生成论文投递版；
3. 若两者同名图差异 > 5%，先回溯口径再写结论。
