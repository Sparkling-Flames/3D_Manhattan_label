# C2: OOD风险代理(d_t)约束规范（2026重写版，详细执行稿）

## 0. 适用范围与目标
本规范用于 Dev C 实现 `tools/compute_dt_score.py`，计算并回写 `merged_all.csv` 的 `d_t` 相关字段，作为 OOD 风险代理。

核心目标：
1. **Cold-start**：在任何人工标注前即可计算。
2. **无泄漏**：不得使用任何人工标注派生信息。
3. **可复现**：固定参考集、固定主参数、固定审计输出。
4. **可披露**：对参考集代表性与失败样本有完整证据链。

---

### 1.1 与 `phase1 (2).html` 对齐
- `d_t` 来源：HOHONet 倒数第二层 embedding + kNN 距离。
- 用途：OOD 风险信号，服务 difficulty stress split 与路由策略。
- 主参数：`K=10`，其余参数用于附录敏感性分析。

### 1.2 与 `merged_all.csv` 对齐

建议补充：

### 1.3 与 `visualize_output_v2.md` 对齐
`d_t` 必须可用于后续分层分析，并保证 NA 状态可解释，不允许“静默补值”影响图表。

### 2.1 白名单（允许）
```text
task_id

```

### 2.2 黑名单（禁止）
```text

```

规则：
- 只要出现黑名单字段参与计算流程，即判定 `leakage_check_failed`。
- 泄漏检查失败不得输出主结果。

## 3. 参考集冻结（Reference Pool Freezing）

### 3.1 冻结规则
1. 数据源固定为 `Calibration_manual`。
2. 按 `task_id` 字典序升序。
3. 取前 100 作为主参考池。
4. 若同一图像存在多个任务版本，按 `base_task_id` 先去重，再取排序后的前 100。
5. 不允许基于图像质量、难度、文件大小做筛选。

### 3.2 `reference_pool_manifest.json` 最小结构
```json
{
  "meta": {
    "strategy": "lexicographical_top_100",
    "source": "Calibration_manual",
    "frozen_at": "2026-03-01T00:00:00Z",
    "ref_hash": "sha256:...",
    "model_version": "hohonet_v2",
    "embedding_backend": "hohonet.penultimate",
    "pool_size": 100,
    "dedup_key": "base_task_id"
  },
  "refs": [
    {
      "task_id": "calib_0001",
      "base_task_id": "scene_0001",
      "image_path": "...",
      "source_split": "Calibration_manual",
      "inclusion_rank": 1,
      "embedding_hash": "sha256:..."
    }
  ]
}
```

最小字段约束：
- `meta` 必须至少含 `strategy`, `source`, `frozen_at`, `ref_hash`, `model_version`, `embedding_backend`, `pool_size`, `dedup_key`。
- `refs[*]` 必须至少含 `task_id`, `base_task_id`, `image_path`, `source_split`, `inclusion_rank`, `embedding_hash`。
- Manifest 中**禁止**出现任何 `quality`, `iou`, `difficulty`, `worker`, `manual_label` 派生字段。
- 同一 `base_task_id` 在 `refs` 中最多出现一次。

### 3.3 代表性披露要求
审计报告中必须包含：
- `scene_type` 分布
- `difficulty` 分布（只做披露，不参与筛选）
- 与完整池的分布差异指标（建议 KL）
- 结论文本：选择过程是否引入明显偏差

## 4. embedding 抽取约束

1. 使用 HOHONet 指定层：`horizon_refinement`。
2. 维度固定：1024。
3. 推理模式：`torch.no_grad()`。
4. 模型参数冻结，不允许训练态更新。
5. 失败图像需返回 NA + 原因，不得重写为默认向量。
6. 主实现**禁止**替代 embedding 后端进入主结果。

### 4.1 主实现与附录稳健性对照的边界
- 主结果仅允许 `HOHONet + fixed reference_pool_manifest.json + K=10 + euclidean + pool_size=100`。
- 若主环境不可用，可在附录中报告替代 embedding 后端的一致性对照，但必须：
  1. 产出独立 manifest/hash；
  2. 写入独立结果文件或独立字段前缀（如 `d_t_alt_*`）；
  3. 明确声明“不进入主结论”。
- 禁止用附录替代结果覆盖主结果字段。

---

## 5. kNN 计算规范

主公式：
$$
d_t = \frac{1}{K}\sum_{i=1}^{K}\|e_t - e_{ref_i}\|_2
$$

### 5.1 主分析参数（锁定）
- `K=10`
- `metric=euclidean`
- `algorithm=brute`
- `n_jobs=1`

### 5.2 附录敏感性参数（仅附录，不进主结论）
- `K in {5, 20}`
- `metric=cosine`
- `pool_size in {50, 150}`

### 5.3 确定性要求
- 同一输入、同一模型版本、同一参考池 hash，输出 `d_t` 小数点后 6 位一致。

---

## 6. 失败熔断与降级策略

### 6.1 失败分类
| 失败类型 | 状态码 | 行为 |
|---|---|---|
| embedding 提取失败 | `extract_fail` | `d_t=NA`，记录原因 |
| 参考集 hash 不匹配 | `ref_hash_mismatch` | 立即中止任务批处理 |
| 维度不匹配 | `embed_dim_error` | `d_t=NA` + 告警 |
| kNN 计算异常 | `knn_runtime_error` | `d_t=NA` + 告警 |

### 6.2 禁止行为
- 禁止把失败样本 `d_t` 填 0。
- 禁止用均值/中位数插补。
- 禁止静默跳过且不记录。

---

## 7. 写回 `merged_all.csv` 的字段契约

| 字段 | 类型 | 规则 |
|---|---|---|
| `d_t` | float/`NA` | 成功时非负；失败时 `NA` |
| `d_t_status` | str | `success`/失败状态码 |
| `d_t_k` | int | 主分析固定 10 |
| `d_t_ref_hash` | str | 必须等于 manifest hash |
| `d_t_model_ver` | str | 模型版本标签 |
| `d_t_metric` | str | 主分析固定 `euclidean` |
| `d_t_pool_size` | int | 主分析固定 100 |
| `d_t_failure_reason` | str/空 | 失败时必填 |
| `d_t_compute_ts` | str | ISO 时间戳 |

补充：
- 若源数据含 `base_task_id`，实现应保留该字段并在审计中用于去重与泄漏检查。
- 同一 `base_task_id` 的多版本任务不得同时进入参考池与待评估集合中的“参考端”。

## 10. 验收清单（必须全部通过）
1. Leakage Check：输入防火墙通过，无 label 派生字段参与。
2. Reference Check：参考池 hash 一致；策略为按 `base_task_id` 去重后按 `task_id` 截断。
3. Determinism Check：重跑 `d_t` 6 位小数一致。
4. Failure Check：失败样本全部 `NA` 且有状态码/原因。
5. Appendix Boundary Check：任何替代 embedding 仅进入附录结果，不覆盖主字段。
6. Disclosure Check：代表性与敏感性边界披露完整。

## 11. 与 C1 / C3 的协作边界
- 与 C1：`model_issue` 来自扰动体系，`d_t` 计算不可读取该字段。
- 与 C3：`difficulty_split.py` 可读取 `d_t`，但 C2 不负责切分策略。
- 与 Dev A：本模块只回写 `d_t*` 字段，不直接修改 `r_u` / `worker_group`。

## 12. 变更管理
若修改主参数（K、metric、pool_size）或参考集策略，必须同时更新：
1. 本规范版本号。
2. `reference_pool_manifest.json` hash。
3. 预注册记录（Week1 lock）。
4. 论文附录敏感性披露。

未同步更新时，结果不得进入主结论。
