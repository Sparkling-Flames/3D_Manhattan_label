### 2. difficulty标签来源口径（必须写死，确保可复现性）
明确difficulty标签的聚合层级和来源，避免不同实现产生不同split：

```python
# difficulty标签来源口径（必须Week1锁定）
difficulty_label_source: {
    "aggregation_level": "task_level_consensus",      # 任务级共识（不是单个worker）
    "consensus_method": "majority_vote",             # 多数投票（简单多数）
    "minimum_annotations": 2,                        # 最少需要2个标注才能形成共识
    "tie_breaking": "random_by_seed",               # 平局时按固定随机种子打破
    "empty_consensus_handling": "exclude_from_split",  # 无法形成共识的任务排除出分层
    "source_field": "difficulty",                    # 直接来自Label Studio的difficulty字段
    "raw_format": "semicolon_separated_aliases",     # 原始格式：分号连接的别名
    "aggregation_timestamp": "pre_split",            # 在split前完成聚合
    "version_lock": "week1_frozen"                  # Week1后锁定，不可更改
}

# 任务级共识形成算法（确定性算法）
def form_task_level_difficulty_consensus(df_task_annotations, seed=42):
    """
    输入：同一task_id的所有标注者的difficulty标签
    输出：单一共识difficulty等级（easy/medium/hard）
    
    算法步骤：
    1. 解析每个标注者的多选标签（分号分割）
    2. 按difficulty_mapping映射到easy/medium/hard
    3. 处理trivial_exclusivity冲突（trivial不能与其他共存）
    4. 多数投票决定最终等级
    5. 平局时按固定随机种子打破
    6. 返回共识等级或None（无法共识）
    """
    
    # 步骤1-3：解析和冲突处理
    all_mapped_levels = []
    for annotator_labels in df_task_annotations['difficulty']:
        levels = annotator_labels.split(';')
        mapped_levels = [difficulty_mapping[level] for level in levels]
        
        # trivial_exclusivity：如果trivial与其他共存，排除trivial
        if 'trivial' in levels and len(levels) > 1:
            mapped_levels = [lvl for lvl in mapped_levels if lvl != 'easy']
        
        all_mapped_levels.extend(mapped_levels)
    
    # 步骤4-5：多数投票 + 固定随机种子打破
    if not all_mapped_levels:
        return None
    
    level_counts = Counter(all_mapped_levels)
    max_count = max(level_counts.values())
    top_levels = [level for level, count in level_counts.items() if count == max_count]
    
    if len(top_levels) == 1:
        return top_levels[0]
    else:
        # 平局时按固定随机种子打破
        rng = random.Random(seed)
        return rng.choice(top_levels)

# Leakage Check执行函数
def perform_leakage_check(train_df, test_df, check_config):
    """
    执行完整的leakage检查
    返回：(is_clean, leakage_report)
    """
    leakage_report = {
        "checks_performed": [],
        "violations_found": [],
        "recommendations": []
    }
    
    # 1. Image ID唯一性检查（必须）
    train_images = set(train_df['image_id'].unique())
    test_images = set(test_df['image_id'].unique())
    overlap_images = train_images.intersection(test_images)
    
    if overlap_images:
        leakage_report["violations_found"].append({
            "type": "image_id_overlap",
            "overlapping_images": list(overlap_images),
            "severity": "critical",
            "description": "同一image_id同时出现在train和test"
        })
    
    # 2. Task ID跨split检查（必须）
    train_tasks = set(train_df['task_id'].unique())
    test_tasks = set(test_df['task_id'].unique())
    overlap_tasks = train_tasks.intersection(test_tasks)
    
    if overlap_tasks:
        # 检查是否是同一task的不同版本（如manual vs semi）
        for task_id in overlap_tasks:
            train_versions = set(train_df[train_df['task_id']==task_id]['condition'])
            test_versions = set(test_df[test_df['task_id']==task_id]['condition'])
            if train_versions != test_versions:
                leakage_report["violations_found"].append({
                    "type": "task_id_version_split",
                    "task_id": task_id,
                    "train_versions": list(train_versions),
                    "test_versions": list(test_versions),
                    "severity": "critical",
                    "description": "同一task_id的不同版本跨split"
                })
    
    # 3. 近重复检查（可选，如果实现）
    # ...（实现基于图像哈希的近重复检测）
    
    # 4. 生成最终报告
    is_clean = len(leakage_report["violations_found"]) == 0
    leakage_report["is_clean"] = is_clean
    leakage_report["leakage_check_passed"] = is_clean
    
    if not is_clean:
        leakage_report["recommendations"] = [
            "必须修复image_id/task_id重叠后才能继续",
            "考虑重新设计split策略确保无重叠",
            "如果存在近重复，考虑排除或统一处理"
        ]
    
    return is_clean, leakage_report
```
