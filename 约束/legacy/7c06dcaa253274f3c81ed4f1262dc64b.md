# C2: OOD风险代理(d_t)约束规范（最终版）

## 概述
本文档定义了基于HOHONet倒数第二层特征向量的OOD(Out-of-Distribution)风险代理d_t的计算约束规范。严格遵循论文方法章节要求：d_t为标注前可得的风险代理(cold-start proxies)，避免循环依赖。本版本补充了代表性说明和敏感性分析边界收紧。

## 核心约束

### 1. 输入约束（标注前可得）
严格遵循"cold-start proxies"原则，输入仅为图像信息，不包含任何人工标注：

### 4. kNN距离计算约束（与论文附录一致 + 确定性保证）
```python
# kNN参数（论文附录K=10，敏感性分析K∈{5,10,20} + 确定性保证）
knn_params: {
    "n_neighbors": 10,                # 主分析K=10
    "metric": "euclidean",           # L2距离（论文规定主度量）
    "algorithm": "ball_tree",        # 确定性算法（确保结果可复现）
    "n_jobs": 1,                     # 单线程执行（避免并行非确定性）
    "deterministic_mode": True       # 强制确定性模式
}

# d_t计算公式（与方法章节一致）
d_t(task) = (1/K) * Σ_{i=1}^K ||e_t(task) - e_t(ref_i)||_2
其中：e_t为HOHONet penultimate embedding，K=10
```
